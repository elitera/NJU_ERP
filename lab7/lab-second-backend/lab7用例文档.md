### 4.1 账户管理

| ID       | 1        | 名称             | 账户管理 |
| -------- | -------- | ---------------- | -------- |

| 创建日期 | 2022.7.7 | 最后一次更新日期 | 2022.7.8 |

| 参与者   | 财务人员，目的是对账户进行管理                               |
| :------- | :----------------------------------------------------------- |
| 触发条件 | 财务人员需要管理公司的银行账户                               |
| 前置条件 | 财务人员必须已经被识别和授权                                 |
| 后置条件 | 系统记录修改后的账户信息                                     |
| 优先级   | 高                                                           |
| 正常流程 | 1. 查询帐户信息<br/>&emsp;1. 财务人员选择查找方式（按帐户编号或者关键字，默认按编号）并输入相关帐户编号或者关键字<br/>&emsp;2. 系统显示该帐户信息，包括帐户的名称和金额<br/>2. 添加客户<br />&emsp;1. 财务人员请求添加新客户<br/>&emsp;2. 系统进入客户信息编辑界面<br/>&emsp;3. 财务人员编辑客户信息并保存<br/>&emsp;4. 系统提示添加成功<br/>3. 修改客户信息<br/>&emsp;1. 财务人员请求更改客户信息<br/>&emsp;2. 系统进入客户信息编辑界面<br/>&emsp;3. 财务人员更改客户信息并保存<br/>&emsp;4. 系统提示更新成功<br/>4. 删除客户<br/>&emsp;1. 财务人员选中待删除客户并选择删除<br/>&emsp;2. 系统询问是否确认删除<br/>&emsp;3. 财务人员确认删除<br/>&emsp;4. 系统完成删除操作并更新客户列表 |
| 扩展流程 | 1-1 a、按编号查询时客户编号格式不正确<br/>&emsp;1、系统提示正确的输入格式并拒绝此次输入<br/>1-1 b、系统中无该客户<br/>&emsp;&emsp;1、系统提示该用户不存在并询问是否添加用户<br/>&emsp;&emsp;&emsp;&emsp;1 a、财务人员选择取消<br/>&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;系统返回客户信息列表<br/>&emsp;&emsp;&emsp;&emsp;1 b、财务人员选择添加<br/>&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;返回正常流程2<br/>2-3 a、客户已存在列表中<br/>&emsp;&emsp;1、系统提示该客户已存在并拒绝执行此次添加<br/>2-3 b、更改客户金额<br/>&emsp;&emsp;1、系统提示客户金额不可修改 |
| 特殊需求 | 1. 账号的查询可以通过输入关键字进行模糊查找<br />2. 仅限最高权限可进行这些操作<br />3. 余额不可修改 |

### 4.2 制定收款单

| ID       | 2        | 名称             | 制定收款单 |
| -------- | -------- | ---------------- | ---------- |
| 创建者   | 任俊宇   | 最后一次更新者   | 任俊宇     |
| 创建日期 | 2022.7.7 | 最后一次更新日期 | 2022.7.8   |

| 参与者   | 财务人员，目标是快速正确地建立收款单或者付款单               |
| :------- | :----------------------------------------------------------- |
| 触发条件 | 财务人员需要记录资金的流动                                   |
| 前置条件 | 财务人员已被系统识别和授权                                   |
| 后置条件 | 系统保存完成的收款单或付款单                                 |
| 优先级   | 高                                                           |
| 正常流程 | 1. 财务人员向系统申请录入收款单<br />2. 系统显示一份空白的收款单 <br />3. 财务人员选择单据类型并录入相关信息<br />4. 系统根据所选的单据类型以及录入的信息生成一张新的单据 <br />5. 财务人员选择向单据里新增一条转账列表 <br />6. 系统显示一条新的转账列表记录 <br />7. 财务人员选择银行账户、输入转账金额和备注 <br />8. 系统显示已输入的转账信息，计算并显示转账总金额，财务人员重复5-8步，直到输入所有转账信息 <br />9. 财务人员确认相关信息并提交 <br />10. 系统提交单据给总经理进行审批 |
| 扩展流程 | 5 a 财务人员需要删除一条转账列表记录 <br />&emsp;1 系统将这条记录删除 <br />7 a 输入转账金额不为正数<br />&emsp;1 系统提示转账金额非法，要求重新输入<br />9 a 财务人员取消确认<br />&emsp;1 系统返回单据编辑界面<br />9 b 转账列表为空<br />&emsp;1 系统拒绝提交并提示转账列表为空 |
| 特殊需求 | 1. 收款单据编号格式为SKD-yyyyMMdd-xxxxx，操作员为当前登录用户 |

| 优先级   | 高                                                           |
| 正常流程 | 1. 查看促销策略<br/>&ensp;&ensp;&ensp;1.总经理选择查看已有促销策略<br/>&ensp;&ensp;&ensp;2. 系统显示已有促销策略的表格<br/>2. 增加促销策略<br/>&ensp;&ensp;&ensp;1.总经理选择新建一条促销策略，填写促销策略的内容<br/>&ensp;&ensp;&ensp;2. 系统保存新建的促销策略单，并生成相应的库存赠送单<br/>3. 更改促销策略<br/>&ensp;&ensp;&ensp;1.总经理选择一条已有的促销策略单<br/>&ensp;&ensp;&ensp;2.系统显示选中的促销策略单的详细内容<br/>&ensp;&ensp;&ensp;3.总经理更改促销策略的内容<br/>&ensp;&ensp;&ensp;4. 系统更新促销策略的状态以及库存赠送单的状态<br/>4. 删除促销策略<br/>&ensp;&ensp;&ensp;1.总经理选择一条已有的促销策略单<br/>&ensp;&ensp;&ensp;2. 系统删除选中的促销策略单，并删除对应的库存赠送单 |
| 扩展流程 | 无                                                           |
| 特殊需求 | 1. 总经理可以针对不同级别的用户制定促销策略（赠品、价格折让、赠送代金劵）<br />2. 总经理可以制定特价包（组合商品降价）<br />3. 总经理可以制定针对不同总价的促销策略（赠品、赠送代金卷）<br />4. 所有促销策略都有起始时间和间隔时间。<br />5. 所有赠品条件促发后，系统会自动建立库存赠送单，由总经理审批通过后，发送消息给库存管理员发放赠品。 |
